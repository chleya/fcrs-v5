# FCRS-v5 深度反思

## 一、问题的本质

### 1.1 为什么会失败？

**现象**: FCRS误差8.79 > Online Learning误差2.11

**根本原因**:

```
我们的假设: 表征竞争 → 更好结果
实际结果: 表征竞争 → 更差结果
```

**哪里错了？**

1. **机制错误**: 竞争不必然带来优化
2. **目标错误**: 我们在优化"维度数量"而非"预测误差"
3. **方法错误**: 阈值判断是人工设计，不是自发的

---

### 1.2 重新审视核心假设

| 假设 | 验证结果 | 修正 |
|------|----------|------|
| H1: 资源约束→稳定 | 需要验证 | - |
| H2: 竞争→选择 | **被证伪** | 竞争需要配合学习 |
| H3: 复杂度→维度 | 需要验证 | - |

---

## 二、技术层面的失败

### 2.1 表征更新的缺失

**问题**: 我们的系统只选择表征，不更新表征

```python
# 当前(错误)
active = pool.select(x)
error = x - active.vector  # 只计算误差
active.fitness_history.append(-error)  # 只记录

# 应该(正确)
active = pool.select(x)
active.vector = active.vector + lr * error  # 更新表征!
```

### 2.2 维度增加的代价

**问题**: 新维度 = 新表征 = 更多信息需要拟合

```
维度↑ → 需要更多数据拟合 → 误差↑
```

---

## 三、正确的方向

### 3.1 结合学习与竞争

```python
class ImprovedFCRS:
    def step(self, x):
        # 1. 选择
        active = self.pool.select(x)
        
        # 2. 学习(关键!)
        error = x - active.vector
        active.vector += self.lr * error  # 在线学习
        
        # 3. 竞争
        if error > threshold:
            self.competition()
```

### 3.2 维度的代价

不是"维度越多越好"，而是:
- 维度增加 = 拟合能力↑
- 维度增加 = 需要更多数据
- 平衡点: 边际收益 = 边际成本
