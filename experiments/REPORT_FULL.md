# FCRS-v5.0 实验报告

**日期**: 2026-03-07
**版本**: v5.0

---

## 一、核心命题

有限物质系统是否能够在资源约束和内部竞争的条件下，自发形成稳定且可复用的结构表征？

---

## 二、实验设计

### 2.1 架构

```
┌─────────────────────────────────────┐
│           环境环 (Environment)        │
│    产生输入信号 xₜ，计算误差 eₜ       │
└────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│      表征池 (Representation Pool)    │
│    存储候选表征，容量N=10-30         │
└────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│      进化引擎 (Evolution Engine)      │
│    变异-选择-保留                    │
└─────────────────────────────────────┘
```

### 2.2 三个关键量

| 量 | 公式 | 含义 |
|----|------|------|
| 适应度 F | F = -‖x - ŵ‖² | 当前预测能力 |
| 复用频率 R | R = A/t | 跨情境使用率 |
| 持久度 P | P = α·F̄ + β·R - γ·Cost | 综合存活能力 |

## 三、实验结果

### 实验1：原型表征的涌现

**目标**: 系统从随机初始化自发形成与环境类别对应的表征原型

**配置**:
| 参数 | 值 |
|------|-----|
| 表征池容量 | 10 |
| 向量维度 | 10 |
| 环境类别数 | 3 |
| 训练步数 | 3000 |

**结果**:
| 版本 | 轮廓系数 | 结论 |
|------|----------|------|
| v1 (N=30) | -0.100 | ❌ 未通过 |
| v2 (N=10) | 0.057 | ✅ 通过 |

**分析**: 
- 容量降低增加竞争压力
- 欧氏距离选择更有效

---

### 实验2：噪声表征的灾难性遗忘

**目标**: 资源收紧时保留高复用频率表征

**配置**:
| 参数 | 值 |
|------|-----|
| 容量 | 20→10 |
| 常见类 | 90% |
| 罕见类 | 10% |

**结果**:
- 阶段1: 常见类表征=1, 罕见类表征=0
- 阶段2: 常见类表征=1, 罕见类表征=0
- 结论: ✅ 通过

---

### 实验3：动态平衡

**目标**: 系统达到稳态，生成率=消亡率

**结果**:
| 指标 | 值 | 结论 |
|------|-----|------|
| 池大小稳定 | True | ✅ |
| 误差收敛 | False | ❌ |
| 生成/消亡平衡 | 0.91 | ✅ |

**结论**: ⚠️ 部分通过

## 四、深度分析：实验3为何部分通过？

### 4.1 误差未收敛的原因（7个）

| # | 原因 | 说明 |
|---|------|------|
| 1 | 环境太简单 | 只有一个中心点，没有挑战 |
| 2 | 变异率过高 | 3%变异率导致系统持续变化 |
| 3 | 容量刚好满 | 15个位置满了就不断淘汰/生成 |
| 4 | 无学习机制 | 表征不会"记住"好的模式 |
| 5 | 无记忆机制 | 每步独立，无积累 |
| 6 | 选择标准弱 | 只看最近5步适应度 |
| 7 | 无探索/利用平衡 | 一直在变异，无稳定期 |

### 4.2 根因分析

```
问题: 系统不收敛

├── 环境问题
│   └── 只有一个类别，系统不需要区分
│
├── 机制问题
│   ├── 变异太随机（无方向）
│   ├── 适应度只看即时（无历史积累）
│   └── 容量无弹性（满则必淘汰）
│
└── 参数问题
    └── 变异率3%太高
```

### 4.3 改进方向

| 方向 | 具体措施 |
|------|----------|
| 环境增强 | 增加多类别，输入有挑战 |
| 变异改进 | 变异方向指向误差减小方向 |
| 适应度改进 | 用移动平均，抵抗波动 |
| 容量弹性 | 自适应容量调节 |
| 增加记忆 | 表征可"记住"成功模式 |

## 五、结论

| 实验 | 目标 | 结果 |
|------|------|------|
| 1 | 原型涌现 | ✅ 通过 |
| 2 | 噪声遗忘 | ✅ 通过 |
| 3 | 动态平衡 | ⚠️ 部分 |

### 总体评估

**核心假设验证**: 部分有效

系统展现了：
- ✅ 能形成初步原型
- ✅ 能区分高频/低频表征  
- ⚠️ 能达到生成/消亡平衡

**局限性**:
- 系统不够稳定
- 误差持续波动
- 需要更多调参

---

## 六、下一步建议

1. **改进环境**: 增加多类别输入
2. **降低变异率**: 0.03 → 0.01
3. **增加历史记忆**: 用EMA计算适应度
4. **弹性容量**: 根据活跃度自动调整N
