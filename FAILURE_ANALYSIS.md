## 一、核心结论

### 假设状态

| 假设 | 状态 |
|------|------|
| "维度扩张 → 泛化提升" | ❌ 未证实 |
| "假设被证伪" | ⚠️ 过早结论 |

### 真正发现

> 扩张**改善了拟合**，但**未稳定结构**

这不等于"扩张无用"，而是：
> 我们的系统**缺少某个关键条件**

---

## 二、为什么扩张没有稳定？

### 可能的解释（H1-H3）

#### H1: 缺乏容量成本

**假说**: 系统没有惩罚维度增长

**证据**:
- 维度从10增长到50
- 没有自然停止机制

**应该添加**:
```
cost = λ * dimension_count
total_loss = task_loss + cost
```

---

#### H2: 任务复杂度不足

**假说**: 任务太简单，不需要结构自适应

**证据**:
- 简单环境(3类): 维度50
- 复杂环境(10类): 也是维度50

**应该测试**:
- 更难的任务（高维、时变、多模态）

---

#### H3: 竞争机制太弱

**假说**: 选择压力不足

**证据**:
- pool=5 但只有1个被高频使用
- 竞争未能淘汰低效表征

**应该加强**:
- 更激进的选择策略
- 更高的淘汰率

---

## 三、哪些机制确实有效？

### 有效机制

| 机制 | 证据 | 价值 |
|------|------|------|
| **竞争筛选** | 适应度高的表征被保留 | ✅ 核心有效 |
| **资源约束** | pool=3最优 (error=1.44) | ✅ 关键有效 |
| **在线学习** | error持续下降 | ✅ 基础有效 |

### 无效/可疑

| 机制 | 证据 | 问题 |
|------|------|------|
| 维度扩张 | 50维过拟合3.22 | ❌ 无容量控制 |
| 对称性破缺 | 触发0次 | ⚠️ 条件太严 |
| 临界状态 | 触发0次 | ⚠️ 检测不准 |

## 六、下一步实验设计

### 实验A: 容量成本测试

**假说**: H1 - 缺乏容量成本

**设计**:
```
System C + λ*dim penalty
λ ∈ {0, 0.001, 0.01, 0.1}
```

**预期**: 某个λ值使维度稳定

---

### 实验B: 任务复杂度测试

**假说**: H2 - 任务太简单

**设计**:
```
E ∈ {3, 5, 10, 20, 50}
测量维度随复杂度变化
```

**预期**: 难任务需要更多维度

---

### 实验C: 竞争强度测试

**假说**: H3 - 竞争太弱

**设计**:
```
淘汰率 ∈ {10%, 30%, 50%, 90%}
```

**预期**: 高淘汰率 → 更稳定的表征
