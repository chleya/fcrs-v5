# 有限竞争表征系统 v5.1

## 核心概念

- **资源有限性**: 系统只能维护N个表征
- **环境闭环**: 通过与环境的持续交互获取反馈
- **内部竞争**: 多个表征为有限的生存资源而竞争

## 三个模块

1. **环境环 (Environment Loop)**
   - 产生输入信号 xₜ
   - 计算预测误差
   - **v5.1新增**: 输出残差信号（未解释的部分）

2. **表征池 (Representation Pool)**
   - 存储候选表征结构
   - 资源有限（容量N）
   - **v5.1新增**: 维度预算管理

3. **进化引擎 (Evolution Engine)**
   - 变异-选择-保留
   - 执行淘汰逻辑
   - **v5.1新增**: 新维度诞生 + 维度竞争

## 三个关键量

1. **适应度 (Fitness)**: 表征对当前输入的预测能力
2. **复用频率 (Reuse)**: 表征在不同情境中被使用的普遍程度
3. **持久度 (Persistence)**: 综合适应度和复用频率的淘汰决策依据
4. **压缩增益 (Compression Gain) [v5.1新增]**: 新维度对减少残差的贡献

## v5.1 新维度机制

### 核心思想

系统在资源约束下不仅能**保持**有限表征，还能**主动扩展**表征空间。

### 三环咬合

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  残差信号   │────>│ 新维度诞生  │────>│ 维度竞争   │
│(环境环输出) │     │(压缩增益阈值)│     │(清理低贡献)│
└─────────────┘     └─────────────┘     └─────────────┘
       ^                                        │
       │                                        │
       └────────────────────────────────────────┘
                    闭环反馈
```

### 新维度诞生规则

```python
# 只有满足以下条件才允许诞生新维度：
1. rep.reuse >= spawn_reuse_threshold  # 复用次数足够高
2. compression_gain > min_compression_gain  # 对残差有真实贡献
3. pool.total_budget >= dim_cost  # 有足够的维度预算
```

**参数**:
- `spawn_reuse_threshold = 5`: 需要至少复用5次
- `min_compression_gain = 0.001`: 压缩增益必须>0.1%
- `dim_cost = 1.0`: 每加1维扣1单位预算

### 维度竞争规则

```python
# 定期（每500步）执行：
for r in pool:
    max_contrib = max(r.dim_contrib)
    useless_dims = r.dim_contrib < max_contrib * 0.05  # 贡献<5%
    r.vector[useless_dims] = 0.0  # 置零
```

### 测试结果

- **300步**: 成功诞生2个新维度
- **压缩增益**: 0.312, 0.033
- **结论**: 机制验证通过
